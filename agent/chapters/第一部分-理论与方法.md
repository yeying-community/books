# 第一部分：理论与方法

这一部分先不急着讨论某个具体项目，而是先解决三个更基础的问题：

- Agent 为什么会出现。
- Agent 和 Workflow、RAG、聊天机器人、Copilot 到底差在哪里。
- 一个真正能长期运行的 Agent，需要面对哪些工程问题。

如果没有这层理论框架，后面无论看 `Claude Code`、`OpenClaw` 还是 `Harness`，都很容易陷入“细节知道不少，但不知道为什么重要”的状态。

因此，本部分的目标不是讲概念热词，而是给读者一套可以反复使用的分析坐标：

- 看一个系统时，先看它的最小工作闭环。
- 分析一个 Agent 时，优先判断上下文、工具、记忆、权限、恢复和评测是怎么组织的。
- 讨论成本时，不只看 `token`，也看延迟、失败率、维护复杂度和系统扩展性。

后面的项目实践部分，都会建立在这套坐标之上。
