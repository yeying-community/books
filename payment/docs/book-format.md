# Payment 书籍结构推荐（v0.1）

## 1. 整本书的骨架

建议按“认知 -> 流程 -> 系统 -> 风险 -> 全球化”的顺序组织，避免读者在细节里迷路。

### Part 0 导读

- 这本书解决什么问题
- 读者是谁，不是谁
- 阅读路径（入门读法 / 架构师读法 / 运营读法）

### Part 1 基础认知（统一词汇）

- 支付网络角色与关系（Cardholder / Merchant / Acquirer / Network / Issuer）
- 卡收单 vs 本地支付（LPM）差异
- 收单模式（PSP / PF / Platform Acquiring）

### Part 2 交易生命周期（核心主线）

- Authorization（授权）
- Clearing（清算）
- Settlement（结算）
- 退款、撤销、拒付与逆向流程

### Part 3 风控与争议（经营安全）

- 准入风控、交易风控、结算风控
- 争议与拒付体系
- 合规与监管约束（KYC/AML/PCI/数据隐私）

### Part 4 全球化与未来（战略层）

- 区域支付差异（欧美、东南亚、拉美）
- 支付编排与多通道路由
- 未来趋势：卡网络、钱包、本地支付、多轨支付

### Part 5 附录（可检索资产）

- 术语表（中英对照）
- 缩写表（如 MCC、BIN、RDR、A2A）
- 常见图谱（交易流、账务流、资金流）
- FAQ（按角色分类：产品 / 风控 / 财务 / 技术）

## 2. 单章固定节奏（建议统一）

每章建议统一为 9 个块，保证可读性和复用性：

1. 本章要解决的问题（1-3 个）
2. 先修知识（读者不会什么就看不懂）
3. 一张图总览（流程图或关系图）
4. 核心概念定义（术语一次讲透）
5. 主流程拆解（按阶段、状态、参与方）
6. 常见异常与误区（至少 3 个）
7. 实战案例（最好含时序和账务结果）
8. 本章总结（3-5 条）
9. 下一章预告（承上启下，明确“下一步问题”）

## 3. 章节内部建议结构

同一章内建议遵守这个展开顺序：

- 先回答“是什么”
- 再回答“为什么”
- 再回答“怎么做”
- 最后回答“失败会怎样”

这样能同时照顾入门读者和实操读者。

## 4. 质量门槛（写完后自检）

- 术语是否前后一致（同一概念不换 3 种叫法）
- 是否把“交易成功”和“资金到账”明确区分
- 是否明确“系统状态”和“财务状态”差异
- 是否给出至少 1 个可执行检查清单（Checklist）
- 是否包含下一章过渡句，保持系列连贯

## 5. 版本化沉淀建议

- 每章顶部保留 `版本 / 更新时间 / 作者 / 审核人`
- 每章底部保留 `变更记录`（新增、修订、删除）
- 大改动时写“为什么改”，不是只写“改了什么”
