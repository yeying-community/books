# 自动发布脚本模板

> 版本：v1.6  
> 更新时间：2026-04-20  
> 用途：为 CI/CD 提供最小可运行的发布门禁脚本骨架

## 1. 使用说明

1. 先对照 [门禁脚本参数手册.md](门禁脚本参数手册.md) 配置参数。
2. 按你们的 CI 平台替换“通知”和“归档”实现细节。
3. 生产发布前至少完成一次回滚演练。

## 2. Shell 模板（可复制）

```bash
#!/usr/bin/env bash
set -euo pipefail

RELEASE_VERSION="${RELEASE_VERSION:-v1.6}"
RULE_VERSION="${RULE_VERSION:-R2.1.0}"
TARGET_PATH="${TARGET_PATH:-docs/acquiring}"
MODE="${MODE:-normal}"
FAIL_ON="${FAIL_ON:-error}"
WARN_THRESHOLD="${WARN_THRESHOLD:-20}"
PILOT_DECISION="${PILOT_DECISION:-GO}"
ROLLBACK_ON_FAIL="${ROLLBACK_ON_FAIL:-true}"
REPORT_OUTPUT="${REPORT_OUTPUT:-docs/acquiring/.reports/latest.md}"
LOG_OUTPUT="${LOG_OUTPUT:-docs/acquiring/.logs/latest.json}"

echo "[INFO] Start release gate: ${RELEASE_VERSION}"

# 1) 文档校验（结构/链接/图谱）
./scripts/validate_docs.sh \
  --target-path "${TARGET_PATH}" \
  --mode "${MODE}" \
  --fail-on "${FAIL_ON}" \
  --warn-threshold "${WARN_THRESHOLD}" \
  --report-output "${REPORT_OUTPUT}" \
  --log-output "${LOG_OUTPUT}"

# 2) 试点结论判定
if [[ "${PILOT_DECISION}" == "HOLD" ]]; then
  echo "[WARN] Pilot decision is HOLD, stop release."
  exit 20
fi

if [[ "${PILOT_DECISION}" == "ROLLBACK" ]]; then
  echo "[ERROR] Pilot decision is ROLLBACK."
  if [[ "${ROLLBACK_ON_FAIL}" == "true" ]]; then
    ./scripts/rollback_release.sh --release-version "${RELEASE_VERSION}"
  fi
  exit 30
fi

# 3) 执行发布
./scripts/publish_docs.sh --release-version "${RELEASE_VERSION}"
echo "[INFO] Release completed: ${RELEASE_VERSION}"
exit 0
```

## 3. CI 集成建议

| 阶段 | 建议动作 | 阻断条件 |
|---|---|---|
| `lint` | 快速语法和链接检查 | 解析失败 |
| `gate` | 规则门禁与试点结论判定 | ERROR / HOLD / ROLLBACK |
| `publish` | 版本发布与索引更新 | 发布脚本异常 |
| `archive` | 报告与日志归档 | 归档失败需告警 |

## 4. 安全建议

1. 生产环境禁止 `FAIL_ON=none`。
2. 强制记录触发人、触发时间、规则版本。
3. 回滚脚本必须可幂等执行。
4. 所有通知失败都需要补偿重试。
