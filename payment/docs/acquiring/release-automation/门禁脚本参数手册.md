# 门禁脚本参数手册

> 版本：v1.6  
> 更新时间：2026-04-20  
> 适用脚本：`release-gate.sh` / `release-gate.ts`

## 1. 参数设计原则

1. 参数命名必须可读、可搜索、可追溯。
2. 关键安全参数必须显式传入，禁止隐式默认放行。
3. 同一参数在本地与 CI 环境语义必须一致。
4. 所有参数变更必须同步更新版本记录与示例命令。

## 2. 核心参数清单

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `--release-version` | string | 是 | 无 | 发布版本，如 `v1.6` |
| `--rule-version` | string | 是 | 无 | 规则版本，如 `R2.1.0` |
| `--target-path` | string | 是 | `docs/acquiring` | 校验与发布目标路径 |
| `--mode` | enum | 否 | `normal` | `normal` / `strict` / `dry-run` |
| `--fail-on` | enum | 否 | `error` | `error` / `warn` / `none` |
| `--warn-threshold` | number | 否 | `20` | WARN 允许阈值（条数） |
| `--allowlist-file` | string | 否 | 空 | 白名单文件路径 |
| `--pilot-decision` | enum | 否 | `GO` | 来自试点结论：`GO/HOLD/ROLLBACK` |
| `--rollback-on-fail` | bool | 否 | `true` | 失败时是否自动回滚 |
| `--report-output` | string | 否 | `.reports/latest.md` | 报告输出路径 |
| `--log-output` | string | 否 | `.logs/latest.json` | 日志输出路径 |
| `--notify-channel` | string | 否 | `docs-oncall` | 通知通道 |
| `--timeout-sec` | number | 否 | `900` | 执行超时秒数 |

## 3. 参数组合建议

| 场景 | 推荐参数组合 |
|---|---|
| 本地预检 | `--mode dry-run --fail-on error --rollback-on-fail false` |
| 试点灰度 | `--mode normal --fail-on error --pilot-decision GO` |
| 高风险变更 | `--mode strict --fail-on warn --warn-threshold 0` |
| 回滚演练 | `--mode normal --pilot-decision ROLLBACK --rollback-on-fail true` |

## 4. 返回码约定（强制）

| 返回码 | 含义 | 处理建议 |
|---|---|---|
| `0` | 发布通过 | 归档并结束流程 |
| `10` | 校验失败（ERROR） | 阻断发布并修复 |
| `20` | 超出 WARN 阈值 | 转 `HOLD` 评审 |
| `30` | 触发回滚 | 检查回滚结果并复盘 |
| `40` | 参数错误 | 修正参数后重试 |
| `50` | 系统异常 | 启动人工接管 |

## 5. 示例命令

```bash
# 场景 1：标准发布
./release-gate.sh \
  --release-version v1.6 \
  --rule-version R2.1.0 \
  --target-path docs/acquiring \
  --mode normal \
  --fail-on error \
  --pilot-decision GO
```

```bash
# 场景 2：严格模式（高风险变更）
./release-gate.sh \
  --release-version v1.6 \
  --rule-version R3.0.0 \
  --mode strict \
  --fail-on warn \
  --warn-threshold 0 \
  --rollback-on-fail true
```

## 6. 常见误配与修正

1. 未传 `--rule-version`：会导致规则追溯中断，必须阻断执行。
2. `--fail-on none` 用于正式发布：高风险，不允许在生产使用。
3. `--warn-threshold` 过大：会掩盖异常波动，建议按团队基线收敛。
4. `--rollback-on-fail false`：仅允许本地 dry-run 使用。
