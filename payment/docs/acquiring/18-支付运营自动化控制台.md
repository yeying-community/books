## 四、控制台目标与边界

### 4.1 核心目标

- 告警去噪：减少无效告警与重复告警。
- 自动分派：按异常类型与责任域自动派单。
- 快速止损：触发预设Runbook执行自动动作。
- 闭环复盘：故障后自动汇总证据并沉淀改进项。

### 4.2 系统边界

- 控制台负责：监控聚合、告警编排、工单分发、运行记录、复盘入口。
- 业务系统负责：执行交易、路由、结算、拒付等具体动作。
- 人工负责：高风险决策、跨团队协调、复盘结论确认。

## 五、推荐能力（MVP）

1. 统一告警中心：按国家、通道、商户分维度聚合异常。
2. 规则编排引擎：支持阈值、组合条件、抑制与去重。
3. 工单自动路由：按问题类型映射到对应Owner队列。
4. Runbook自动执行：低风险问题自动执行标准处置动作。
5. 值班体系：按时区值班并支持跨班交接与升级。
6. 复盘中心：自动汇总故障时间线、影响范围、处置动作。

## 六、统一治理规则（强制）

### 6.1 告警分级

- `P0`：资金安全、批量漏账/重账、核心支付不可用。
- `P1`：大范围成功率下降、结算延迟、拒付异常飙升。
- `P2`：局部国家/商户/通道异常。

### 6.2 自动动作边界

- 允许自动动作：降级、限流、重试、切备。
- 禁止自动动作：大规模冻结资金、跨区域规则禁用、人工审批动作绕过。
- 所有自动动作必须具备回滚键与审计日志。

### 6.3 运行SLA（建议）

| 项目 | 目标SLA |
|---|---|
| P0 首次响应 | 15分钟 |
| P1 首次响应 | 30分钟 |
| P2 首次响应 | 2小时 |
| 工单自动分派成功率 | >= 98% |
| Runbook自动执行成功率 | >= 95% |

## 七、关键指标（建议）

| 指标 | 定义 | 目标方向 |
|---|---|---|
| 告警噪音率 | 无需动作告警 / 总告警 | 下降 |
| 告警去重率 | 去重后告警 / 原始告警 | 上升 |
| 自动分派命中率 | 分派正确工单 / 自动分派工单 | 上升 |
| MTTA | 平均响应时间 | 下降 |
| MTTR | 平均恢复时间 | 下降 |
| Runbook命中率 | 使用Runbook处置工单 / 总工单 | 上升 |

## 九、本章总结

- 运营自动化控制台是“执行中枢”，不是单纯监控面板。
- 自动化价值来自标准化动作和可追踪闭环，而不是告警数量。
- 没有Runbook和复盘机制，自动化会退化为告警堆积。
