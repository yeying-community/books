## 四、监管知识图谱建模边界（强制）

1. 监管实体：法规、条款、监管机构、生效日期、适用区域。
2. 控制实体：流程控制点、审计字段、系统校验规则、报送义务。
3. 任务实体：整改任务、负责人、截止时间、依赖关系、验收标准。
4. 证据实体：审批记录、提交回执、测试结果、对外沟通记录。

## 五、实体关系最小模型

| 关系 | 说明 | 示例 |
|---|---|---|
| `Regulation -> hasArticle -> Article` | 法规包含条款 | PSD2 -> Article 97 |
| `Article -> imposes -> Obligation` | 条款定义义务 | 强认证义务 |
| `Obligation -> mappedTo -> ControlPoint` | 义务映射到控制点 | 风控校验、审计字段 |
| `ControlPoint -> implementedBy -> SystemModule` | 控制点由系统模块实现 | 交易风控引擎 |
| `Gap -> resolvedBy -> Task` | 差距由任务闭环 | 新增字段、改造流程 |
| `Task -> ownedBy -> Team` | 任务归属团队 | 合规、产品、工程 |

## 六、任务自动编排主流程

```mermaid
flowchart LR
  A[输入新规条款] --> B[图谱实体抽取与映射]
  B --> C[查询受影响控制点]
  C --> D[生成差距与任务草案]
  D --> E[按优先级与依赖自动编排]
  E --> F[分配Owner并设置SLA]
  F --> G[周度追踪与升级闭环]
```

图说明：

- 输入：新规条款、当前图谱、历史整改记录。
- 处理：关系推理、差距识别、任务拆分、优先级排序、SLA绑定。
- 输出：可执行任务清单、责任分配、升级路径和验收标准。

## 七、优先级与SLA策略（建议）

| 优先级 | 触发条件 | 目标时限 | 升级规则 |
|---|---|---|---|
| `P0` | 强制新规 + 高罚款风险 | 24 小时确认方案 | 每日升级到负责人 |
| `P1` | 跨系统流程改造 | 5 个工作日完成计划 | 逾期升级到平台Owner |
| `P2` | 字段/报表口径补齐 | 10 个工作日 | 周会跟踪 |
| `P3` | 文档与培训更新 | 15 个工作日 | 月度复盘 |

## 八、自动编排落地要点

1. 任务粒度统一：任务最小粒度必须可验收、可回滚。
2. 依赖显式化：跨团队任务必须有前置依赖和阻塞原因字段。
3. 风险前置化：`P0/P1` 任务必须绑定风险等级和应急预案。
4. 证据自动化：每个任务完成自动挂接测试与审批证据链接。

## 九、最小输出物

- 监管知识图谱快照（实体数、关系数、版本号）。
- 规则到任务映射结果（含优先级、Owner、SLA）。
- 自动编排执行板（待办/进行中/已完成/阻塞）。
- 周度风险清单（逾期任务、潜在违规、应急动作）。

## 十、提交前检查清单

- [ ] 已完成本期监管条款实体抽取与关系映射
- [ ] 已完成差距查询并生成任务草案
- [ ] 已为 `P0/P1` 任务绑定SLA和升级路径
- [ ] 已定义任务验收标准和证据挂接规则
- [ ] 已完成一次跨团队联调演练

## 十一、本章总结

- 监管知识图谱是“规则机器可读”的基础。
- 自动编排让整改任务从“人工拉群”升级为“系统调度”。
- 只有把任务、SLA、证据三者绑定，监管响应才能稳定复用。
