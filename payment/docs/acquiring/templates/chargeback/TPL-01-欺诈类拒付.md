# 模板 01：欺诈类拒付（Fraud Dispute）

## 1. 适用场景

- 非持卡人交易（盗刷）
- 持卡人否认交易真实性

## 2. 关键抗辩思路

- 证明交易由真实用户或受信设备发起。
- 证明交易过程具备有效认证或风险校验。
- 证明交易后行为与历史一致（登录、收货、服务使用）。

## 3. 必备证据

| 编号 | 证据项 | 必须/可选 | 说明 |
|---|---|---|---|
| E01 | 交易授权成功记录 | 必须 | 含授权码、响应码 |
| E02 | 3DS/认证结果 | 建议 | 含认证状态与时间 |
| E03 | 设备指纹与IP信息 | 必须 | 含设备ID、IP地理位置 |
| E04 | 登录与下单行为日志 | 必须 | 登录、加购、支付路径 |
| E05 | 历史交易相似性 | 可选 | 与历史地址、设备一致性 |
| E06 | 履约/交付记录 | 建议 | 数字服务激活或物流签收 |

## 4. 事实陈述模板

```text
本交易于 {time_utc} 发起并于 {time_utc} 授权通过（见 E01）。
交易设备为 {device_id}，IP 为 {ip}，与账户历史行为 {一致/部分一致}（见 E03/E04）。
如适用，本交易通过 {3DS类型} 认证，认证结果为 {result}（见 E02）。
交易完成后，客户账户于 {time_utc} 出现 {登录/使用/下载/激活} 行为（见 E04/E06）。
综上，本交易具备真实用户行为链条，非典型盗刷模式。
```

## 5. 反例风险提示

- 设备与历史完全不一致且无额外认证。
- 同IP高频失败后单笔成功，疑似撞库。
- 履约证据缺失，无法形成闭环。
