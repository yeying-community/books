# 模板 02：未收到货拒付（Goods Not Received）

## 1. 适用场景

- 持卡人主张未收到实体商品。
- 持卡人主张未收到数字服务或激活权益。

## 2. 关键抗辩思路

- 证明商户已按约定完成发货或交付。
- 证明收货方与下单方关联成立。
- 证明交付时间与承诺时效匹配或有合理说明。

## 3. 必备证据

| 编号 | 证据项 | 必须/可选 | 说明 |
|---|---|---|---|
| E01 | 订单详情与承诺时效 | 必须 | 商品、地址、预计时效 |
| E02 | 物流发货凭证 | 必须 | 运单号、发货时间 |
| E03 | 物流签收证明 | 必须 | 签收人、签收时间 |
| E04 | 客户地址与订单地址一致性 | 建议 | 地址匹配截图 |
| E05 | 客户沟通记录 | 建议 | 催件、补发、投诉 |
| E06 | 服务交付日志（数字服务） | 必须（数字） | 账号激活、使用记录 |

## 4. 事实陈述模板

```text
订单 {merchant_order_id} 于 {ship_time_utc} 完成发货（见 E02）。
物流单号 {tracking_no} 显示于 {deliver_time_utc} 由 {receiver} 签收（见 E03）。
订单地址与持卡人提供地址 {一致/高度一致}（见 E01/E04）。
若为数字服务，服务于 {activate_time_utc} 激活并产生使用行为（见 E06）。
综上，商户已履行交付义务。
```

## 5. 反例风险提示

- 仅有发货记录无签收记录。
- 物流轨迹中断且无补救措施说明。
- 数字服务无激活或使用日志。
