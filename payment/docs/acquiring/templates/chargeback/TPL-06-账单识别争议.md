# 模板 06：账单识别争议（Unrecognized Descriptor）

## 1. 适用场景

- 持卡人主张账单商户名无法识别。
- 持卡人主张扣款描述与下单品牌不一致。

## 2. 关键抗辩思路

- 证明账单描述与商户主体存在可解释映射关系。
- 证明下单页或确认邮件明确展示扣款主体信息。
- 证明商户提供了可联系的客服入口与响应记录。

## 3. 必备证据

| 编号 | 证据项 | 必须/可选 | 说明 |
|---|---|---|---|
| E01 | 账单描述配置截图 | 必须 | Descriptor 配置 |
| E02 | 品牌与主体映射说明 | 必须 | 品牌名与法主体 |
| E03 | 下单确认页/邮件 | 必须 | 扣款主体展示 |
| E04 | 客服可达证明 | 建议 | 官网、电话、邮箱 |
| E05 | 客户咨询与回复记录 | 建议 | 已解释账单信息 |

## 4. 事实陈述模板

```text
本交易账单描述为 {descriptor}（见 E01）。
该描述对应商户主体 {legal_entity}，与品牌 {brand_name} 存在映射关系（见 E02）。
客户在下单确认页/确认邮件中可见扣款主体信息（见 E03）。
商户提供了可联系渠道并对客户疑问进行了回复（见 E04/E05）。
综上，本次扣款可被合理识别，不属于未知商户扣款。
```

## 5. 反例风险提示

- Descriptor 与品牌完全无关联且无说明。
- 下单流程没有展示扣款主体。
- 客服渠道不可用导致持卡人直接发起拒付。
