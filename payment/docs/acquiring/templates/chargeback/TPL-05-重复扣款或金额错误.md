# 模板 05：重复扣款或金额错误（Duplicate/Incorrect Amount）

## 1. 适用场景

- 持卡人主张同一订单被重复扣款。
- 持卡人主张扣款金额与订单金额不一致。

## 2. 关键抗辩思路

- 证明多笔交易是否对应不同履约或分次请款。
- 证明金额变更有明确规则与客户确认。
- 若确属系统错误，证明已主动退款并完成补救。

## 3. 必备证据

| 编号 | 证据项 | 必须/可选 | 说明 |
|---|---|---|---|
| E01 | 原始订单金额与币种 | 必须 | 下单金额依据 |
| E02 | 授权与Capture明细 | 必须 | 每笔金额与时间 |
| E03 | 分次请款规则/业务说明 | 建议 | 酒店、租车等场景 |
| E04 | 客户确认记录 | 建议 | 二次扣款确认 |
| E05 | 退款记录（如已补救） | 必须（有补救） | 退款时间与金额 |

## 4. 事实陈述模板

```text
订单金额为 {order_amount} {currency}（见 E01）。
交易流水显示 {n} 笔扣款，分别为 {amount_list}（见 E02）。
多笔扣款原因为 {分次请款/追加消费/系统异常已修复}（见 E03）。
如存在误扣，商户已于 {refund_time_utc} 完成退款 {refund_amount}（见 E05）。
综上，本案属于 {合理分次请款/已完成补救}。
```

## 5. 反例风险提示

- 无法解释第二笔扣款来源。
- 分次请款场景没有规则披露。
- 错扣后未及时退款。
