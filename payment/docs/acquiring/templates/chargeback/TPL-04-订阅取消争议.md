# 模板 04：订阅取消争议（Subscription Cancellation）

## 1. 适用场景

- 持卡人主张已取消订阅但仍被扣款。
- 持卡人主张自动续费条款不清晰。

## 2. 关键抗辩思路

- 证明订阅条款披露清晰且用户确认。
- 证明取消入口可用且取消流程可达。
- 证明本次扣款发生时订阅状态仍有效。

## 3. 必备证据

| 编号 | 证据项 | 必须/可选 | 说明 |
|---|---|---|---|
| E01 | 订阅条款与续费说明快照 | 必须 | 包含续费周期和价格 |
| E02 | 用户勾选同意记录 | 必须 | 同意时间与IP |
| E03 | 订阅状态变更日志 | 必须 | 创建、续费、取消时间 |
| E04 | 取消入口可达证明 | 建议 | 入口截图/流程录屏 |
| E05 | 取消申请记录 | 必须（有申请） | 工单或系统日志 |
| E06 | 扣款时间与状态比对 | 必须 | 扣款时是否仍有效 |

## 4. 事实陈述模板

```text
用户于 {subscribe_time_utc} 创建订阅并同意续费条款（见 E01/E02）。
在本次扣款 {charge_time_utc} 时，订阅状态为 {active}（见 E03/E06）。
若用户提交取消申请，申请时间为 {cancel_time_utc}，生效规则为 {rule}（见 E05）。
取消入口在 {platform} 可正常访问（见 E04）。
综上，本次扣款符合订阅条款与系统状态。
```

## 5. 反例风险提示

- 条款披露不清或无法证明用户明确同意。
- 取消入口不可用或响应异常。
- 取消已生效仍继续扣款。
