# CA-02 取证流水线

## 1. 数据源映射

| 证据类型 | 来源系统 | 拉取方式 | 超时阈值 |
|---|---|---|---|
| 交易证据 | Payment Core | API | 30秒 |
| 履约证据 | OMS/物流 | API/文件 | 2分钟 |
| 沟通证据 | CRM | API | 1分钟 |
| 账单证据 | Billing | API | 30秒 |

## 2. 编排步骤

1. 案件创建后并行触发证据任务。
2. 证据聚合器进行字段标准化与脱敏。
3. 质量检查器执行完整性与一致性校验。
4. 输出证据包版本并写入审计日志。

## 3. 失败处理

- 单数据源超时：重试 2 次后降级人工补录。
- 数据冲突：自动标记 `REVIEW_REQUIRED`。
- 证据缺失：阻断自动提交，进入人工队列。

## 4. 必填审计字段

- case_id
- evidence_pack_version
- source_system
- fetch_time_utc
- rule_version
