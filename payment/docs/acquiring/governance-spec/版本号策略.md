# 规则版本号策略

> 版本：v1.4  
> 更新时间：2026-04-20

## 1. 版本格式

- 统一格式：`R{major}.{minor}.{patch}`。
- 示例：`R2.0.0`、`R2.1.0`、`R2.1.3`。

## 2. 升级规则

| 类型 | 触发条件 | 兼容性要求 | 发布约束 |
|---|---|---|---|
| `major` | 规则语义变化、默认阻断策略变化 | 可能不兼容 | 必须灰度与回滚预案 |
| `minor` | 新增规则或增强检测项 | 应保持兼容 | 必须提供影响说明 |
| `patch` | 误报修复、阈值微调、描述修正 | 必须兼容 | 可快速发布 |

## 3. 标签约定

| 标签 | 含义 | 使用场景 |
|---|---|---|
| `ACTIVE` | 当前生效版本 | 生产默认版本 |
| `CANDIDATE` | 候选版本 | 灰度或试点阶段 |
| `DEPRECATED` | 废弃待下线版本 | 保留兼容窗口 |
| `RETIRED` | 已退役版本 | 不再允许新发布引用 |

## 4. 生命周期建议

1. `CANDIDATE`：至少跑完 1 个发布周期观测。
2. `ACTIVE`：通过评审后切换为默认版本。
3. `DEPRECATED`：发布公告后保留至少 2 周。
4. `RETIRED`：完成迁移并归档后退役。

## 5. 变更记录约束

- 每次版本升级必须记录变更摘要、影响范围、回滚点。
- `major` 升级必须绑定兼容性矩阵与演练记录。
- 无变更记录的版本号不得进入发布导航。
