# 输出约定

## 1. 输出路径规则

- 章节文档：`docs/acquiring/{doc_id}-{title}.md`
- 模板文档：`docs/acquiring/templates/{domain}/...`
- 附件规范：`docs/acquiring/{domain}/...`

## 2. 文档骨架规则

- 必须包含头部元信息：版本、更新时间、作者、审核。
- 必须包含最小章节：问题、先修知识、总结、变更记录。
- 如含图谱，必须附“输入/处理/输出”三行说明。

## 3. 覆盖策略

- `safe_append`：仅补缺失块，不覆盖现有内容。
- `force_replace`：覆盖指定块，必须显式指定块名。
- `dry_run`：只输出变更预览，不落文件。

## 4. 生成后校验

1. markdown 链接校验。
2. 术语可检索校验（至少命中术语表一次）。
3. Mermaid 语法校验（如包含图谱）。
4. 章节结构完整性校验。

## 5. 生成日志建议字段

- `run_id`
- `timestamp_utc`
- `input_hash`
- `output_files`
- `mode`
- `validator_result`
