# 自动提醒策略模板

> 版本：v1.9  
> 更新时间：2026-04-21  
> 用途：配置报送任务的分阶段提醒与升级策略

## 1. 策略基础信息

| 字段 | 内容 |
|---|---|
| 策略版本 | `{{NTF-1.0}}` |
| 适用范围 | `{{EU/APAC/US}}` |
| 通知渠道 | `{{邮件/IM/工单}}` |
| 维护人 | `{{name}}` |

## 2. 分阶段提醒规则

| 阶段 | 触发时点 | 通知对象 | 通知内容 | 升级条件 |
|---|---|---|---|---|
| `T-7` | 截止前 7 天 | 责任人 | 准备提醒 | 未确认接收 |
| `T-3` | 截止前 3 天 | 责任人 + 复核人 | 复核提醒 | 任务未进入复核 |
| `T-1` | 截止前 1 天 | 责任人 + 管理者 | 强提醒 | 仍未完成审批 |
| `T+1` | 逾期 1 天 | 合规负责人 | 逾期升级 | 触发应急流程 |

## 3. 消息模板（示例）

```text
[监管报送提醒] 任务 {{task_id}} 将于 {{deadline}} 截止。
当前状态：{{status}}，责任人：{{owner}}。
请在 {{next_checkpoint}} 前完成 {{required_action}}。
```

## 4. 失败补偿策略

| 场景 | 补偿动作 | 最大重试次数 |
|---|---|---|
| 通知发送失败 | 切换备用渠道并重发 | 3 |
| 接收未确认 | 自动升级至复核人 | 2 |
| 任务状态未更新 | 自动创建跟进工单 | 1 |

## 5. 观测指标

| 指标 | 定义 | 目标 |
|---|---|---|
| 提醒送达率 | 成功送达数 / 总提醒数 | >= 99% |
| 提醒确认率 | 已确认提醒 / 已送达提醒 | >= 95% |
| 逾期率 | 逾期任务 / 总任务 | 下降 |
| 升级处理及时率 | SLA 内处理升级任务 / 升级任务总数 | 上升 |
