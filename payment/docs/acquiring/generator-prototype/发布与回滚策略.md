# 模板生成器发布与回滚策略

> 版本：v1.3  
> 更新时间：2026-04-20

## 1. 发布策略

### 1.1 发布分层

| 层级 | 适用范围 | 策略 |
|---|---|---|
| Dev | 本地验证 | 允许 `WARN`，不阻断 |
| Staging | 试点章节 | `ERROR` 阻断，`WARN` 需记录 |
| Prod | 正式发布 | `ERROR` 阻断，关键 `WARN` 升级审批 |

### 1.2 发布前门禁

1. 结构校验通过。
2. 链接校验通过。
3. 图谱校验无 `ERROR`。
4. 生成日志与报告已归档。
5. 覆盖策略与备份策略已确认。

## 2. 回滚策略

### 2.1 触发条件

- 发布后发现关键章节被错误覆盖。
- 发布后发现规则缺陷导致大面积误报。
- 发布门禁逻辑异常导致错误放行。

### 2.2 回滚步骤

1. 定位发布批次（报告ID、日志ID、提交ID）。
2. 从备份目录恢复受影响文件。
3. 重跑校验确认恢复结果。
4. 暂停新发布并记录事故。
5. 完成根因分析后再恢复发布。

## 3. 回滚演练建议

| 演练项 | 频率 | 验收标准 |
|---|---|---|
| 单文件回滚 | 每周 | 10 分钟内恢复并通过校验 |
| 批量文件回滚 | 每月 | 30 分钟内恢复并输出报告 |
| 门禁异常应急 | 每月 | 负责人可执行降级策略 |

## 4. 风险与控制

| 风险 | 控制措施 |
|---|---|
| 误用 `force_replace` | 增加审批与双人确认 |
| 备份缺失 | 发布前强制校验备份存在 |
| 回滚后遗漏复验 | 回滚流程内置复验步骤 |
| 报告不可追溯 | 强制报告ID与发布批次绑定 |
