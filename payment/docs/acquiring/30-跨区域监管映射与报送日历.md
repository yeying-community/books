## 四、监管映射模型（强制）

### 4.1 映射维度

1. 区域维度：国家/地区、监管机构、适用业务范围。
2. 义务维度：报送类型、频率、截止日、签署要求。
3. 数据维度：字段清单、口径版本、数据源系统。
4. 责任维度：负责人、复核人、升级路径。

### 4.2 映射输出要求

- 每条监管义务必须有唯一映射ID。
- 每条映射必须可追溯到法规条款或监管通知。
- 每条映射必须绑定报送日历和提醒策略。

## 五、报送日历设计

```mermaid
flowchart LR
  A[监管义务映射] --> B[生成报送任务]
  B --> C[任务分配与复核]
  C --> D[到期前提醒]
  D --> E[提交与回执留存]
  E --> F[逾期检查与升级]
```

图说明：

- 输入：监管映射规则、报送频率、责任人配置。
- 处理：任务生成、提醒触发、提交留痕、逾期升级。
- 输出：可执行报送日历、回执证据、升级记录。

## 六、自动提醒机制（建议）

| 提醒阶段 | 时间点 | 通知对象 | 动作 |
|---|---|---|---|
| `T-7` | 截止日前 7 天 | 责任人 | 首次提醒，确认准备状态 |
| `T-3` | 截止日前 3 天 | 责任人 + 复核人 | 二次提醒，确认材料完整性 |
| `T-1` | 截止日前 1 天 | 责任人 + 管理者 | 强提醒，未完成需升级 |
| `T+1` | 逾期 1 天 | 管理者 + 值班 | 触发逾期升级与补救流程 |

## 七、跨区域差异治理

1. 统一主口径：核心指标定义统一，区域差异通过附加字段表达。
2. 差异白名单：允许区域例外，但必须记录批准人和生效期。
3. 版本策略：法规变更导致口径变更时必须升版并公告。
4. 复核机制：跨区域报送前进行一次集中一致性复核。

## 八、风险与控制

| 风险 | 触发场景 | 控制措施 |
|---|---|---|
| 错报 | 字段映射错误 | 字段校验 + 双人复核 |
| 漏报 | 日历缺任务 | 自动生成任务 + 周期对账 |
| 迟报 | 提醒失效或流程阻塞 | 多级提醒 + 逾期升级 |
| 口径漂移 | 区域自定义未同步 | 口径版本治理 + 审计检查 |

## 九、提交前检查清单

- [ ] 已建立跨区域映射矩阵并分配映射ID
- [ ] 已生成覆盖全区域的报送日历
- [ ] 已配置 T-7/T-3/T-1/T+1 自动提醒
- [ ] 已定义逾期升级路径与责任人
- [ ] 已完成一次跨区域报送演练

## 十、本章总结

- 监管映射是“规则可执行”的前提，报送日历是“执行可持续”的保障。
- 自动提醒不是提醒次数，而是要能驱动责任闭环。
- 统一口径与差异治理并存，才能支持全球化合规运营。
