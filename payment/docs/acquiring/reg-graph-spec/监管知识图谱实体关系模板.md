# 监管知识图谱实体关系模板

> 用途：统一沉淀法规、条款、控制点、系统模块与责任团队关系，作为自动差距分析和任务编排输入。

## 1. 版本信息

| 字段 | 示例 |
|---|---|
| 图谱版本 | `RG-2026Q2-v1` |
| 生效日期 | `2026-04-21` |
| 维护团队 | `Compliance Platform` |
| 关联章节 | [32-监管知识图谱与响应任务自动编排.md](../32-监管知识图谱与响应任务自动编排.md) |

## 2. 实体定义

| 实体类型 | 必填字段 | 说明 |
|---|---|---|
| `Regulation` | `id`、`name`、`region`、`effective_date` | 法规或监管通知 |
| `Article` | `id`、`regulation_id`、`article_no`、`summary` | 法规条款 |
| `Obligation` | `id`、`article_id`、`obligation_type` | 合规义务 |
| `ControlPoint` | `id`、`control_name`、`owner_team` | 控制点定义 |
| `SystemModule` | `id`、`module_name`、`repo` | 系统模块 |
| `Task` | `id`、`priority`、`owner`、`due_date` | 整改任务 |

## 3. 关系定义

| 源实体 | 关系 | 目标实体 | 必填字段 |
|---|---|---|---|
| `Regulation` | `hasArticle` | `Article` | `relation_id`、`source` |
| `Article` | `imposes` | `Obligation` | `risk_level` |
| `Obligation` | `mappedTo` | `ControlPoint` | `coverage_status` |
| `ControlPoint` | `implementedBy` | `SystemModule` | `system_owner` |
| `Gap` | `resolvedBy` | `Task` | `deadline` |
| `Task` | `ownedBy` | `Team` | `primary_owner` |

## 4. 快照输出示例（YAML）

```yaml
graph_version: RG-2026Q2-v1
regulation:
  id: REG-EU-PSD2-2026-01
  region: EU
  effective_date: 2026-07-01
relations:
  - from: ARTICLE-97
    type: imposes
    to: OBL-SCA-01
    risk_level: L3
  - from: OBL-SCA-01
    type: mappedTo
    to: CP-RISK-3DS
    coverage_status: partial
```

## 5. 提交前检查清单

- [ ] 实体ID在全图谱范围内唯一
- [ ] 关系类型使用标准枚举，不允许自由文本
- [ ] 每条义务至少映射一个控制点
- [ ] 每个高风险差距已关联任务实体
