# 规则到任务映射模板

> 用途：将监管条款差距自动映射为可执行任务，统一优先级、责任归属和验收口径。

## 1. 输入信息

| 字段 | 示例 | 说明 |
|---|---|---|
| 规则ID | `REG-US-CFPB-2026-04` | 对应法规或监管通知 |
| 条款ID | `ART-12` | 图谱条款节点 |
| 差距ID | `GAP-0007` | 自动差距分析输出 |
| 风险等级 | `L1/L2/L3` | 合规风险评估结果 |
| 影响系统 | `risk-engine, settlement` | 受影响模块 |

## 2. 任务生成表

| 任务ID | 任务名称 | 优先级 | Owner | 截止日期 | 依赖 | 验收标准 |
|---|---|---|---|---|---|---|
| `TASK-001` | 更新风控校验规则 | `P0` | 风控平台 | `2026-04-25` | 无 | 回归通过，误拒率不升高 |
| `TASK-002` | 补齐审计字段 | `P1` | 数据平台 | `2026-04-28` | TASK-001 | 字段落库并可追溯 |
| `TASK-003` | 更新监管报表口径 | `P2` | 合规运营 | `2026-04-30` | TASK-002 | 报表校验通过 |

## 3. 优先级映射规则

| 风险等级 | 任务优先级 | 默认SLA |
|---|---|---|
| `L3` | `P0` | 24 小时内启动 |
| `L2` | `P1` | 5 个工作日完成 |
| `L1` | `P2/P3` | 10-15 个工作日完成 |

## 4. 自动编排输出示例（JSON）

```json
{
  "rule_id": "REG-US-CFPB-2026-04",
  "gap_id": "GAP-0007",
  "tasks": [
    {
      "task_id": "TASK-001",
      "priority": "P0",
      "owner": "risk-platform",
      "due_date": "2026-04-25",
      "acceptance": "qa_passed_and_audit_linked"
    }
  ]
}
```

## 5. 提交前检查清单

- [ ] 每个差距项都已生成至少 1 个任务
- [ ] `P0/P1` 任务均已绑定升级责任人
- [ ] 任务验收标准为可验证结果而非描述性文本
- [ ] 任务与图谱实体关系可追溯
